World Wide Web
It was mainly developed by Tim Berners Lee's. Here's a little timeline.
1988: First IP Connection between North america and europa
1089: a proposal for a System called Mesh which described a more elaberate information management based on links in a readable text/hyperlinks
1990: Hypertextproject WorldWideWeb -> a web of hypertext-documents which are viewed by browsers
1993: the World Wide Web got free to anyone and the Mosaic Webbrowser was released
The internet and the World Wide Web are not the same thing!
There are a couple of reasons why the World Wide Web was such a success. One of them is the fact, that the Internet and Hypertext were married for the first time. Also these three technologies were essential for the development of the World Wide Web as we know it.
The URL (uniform resource locator) and URI(uniform resource identifier), or former UDI (universal document identifier) is a system for (globally) unique identifiers for resources on the Web
HTML aka Hypertext Markup L>anguage
HTTP aka The Hypertext Transfer Protocol
It also had unidirectional links rather than bidirectional links
More infos to the URL:
It is also called web adress, even though this is just colloquially, and it is a reference to a (web) resource that specifies a location on a computer network and it's a mechanism for calling this resource up.
The URL is often confused with the URI(Uniform Resource Identifier) even thoug it's just a specific form of an URI.
The syntax can be very simple or a little bit complicated, it completely depends on the specifix URL. For example it could be [scheme]:[path] which is basically the simplest form.
Now you can build on this our "own" URL. After the [scheme]: you can add "//[host]" and after [path] "?[query]"so it would be [scheme]://[host][path]?[query]. But up on this you can build even more stuff.
After the "//" we can add "userinfo@" and after the "[host]" you can add ":[port]" so it finally would be [scheme]://[userinfo]@[host]:[port][path]?[query]
Some possibles schemes are http, https, ftp, mailto and so on.

Web Server
A webserver shall store, process and deliver webpages to clients. The communication between the server and the client takes place using http. Mostly delivered are HTML Documents.
A web browser or a web crawler starts to communicate by making a request for a specific resource using HTTP or HTTPS. The server responds with the resource or an error.
But content can also be received from clients. This is used e.g. for web forms.
A lot of generic web servers nowadays also support server side scripting using a couple of scripting languages like PHP.

Web Client
Also known as Web Browser, a web client is an application for accessing information on the world wide web. It fetches information resources and displays them to an end device like a computer.
The process is the following: The user inputs a URL, the rendering engine of the browser renders it and then displays it.

About Doctype
The <!DOCTYPE> on the head of a html document exists to show the browser the version of html that is used. It's not always required but you should still use it always.
About the body element
The <body> element contains all of the contents such as text, images and other.
About the em element
The <em> tag is used to show important text that should be higlighted from the rest of the text.
About the head element
The <head> element contains all of the metadata and things like the <style> tag, the <script> tag or the <link> tag.
About the html tag
The <html> element shows the browser that the following document is a html document. It also is the root for all of the other tags and elements of the html document.
About the strong element
The <strong> exists for basically the same reason as the <em> tag. It just has a other style, it's kinda bold, which is also very useful sometimes.
About the title element
The <title> element is required in a html document, it is basically just the title of the document. It is between the <head> tags.

By the way. I managed to use the tags in the text with &lt (<) or &gt (>).
Generally you can make special characters (that mean something else in HTML like < and &) by using &# and then the Hexadecimal code for that specific character.
It is also important to put a ; at the end of the code, so the browser notices, that this is not a colorcode but a HTML Code.

Headings
Special things bout headings: Browsers automatically create some white space before and after the heading. You should use them for Headings only, if you wanna highlight a text you can either make it strong or style it with the style attribute.
Of course you can style a heading with the style attribute. An <hr> tag defines a thematic break in an HTML page, and is most often displayed as a horizontal rule. And it's basically just a fucking line.
Paragraphs
The HTML <pre> element defines preformatted text. The text inside a <pre> element is displayed in a fixed-width font (usually Courier), and it preserves both spaces and line breaks

Links
There are absolute and releative links, absolute Links are links to a completely new website, relative Links are just e.g. HTML Files on the same website, so these are local Links
Head/Meta
The meta in the head can be a couple of different things. e.g. the charset="utf-8" or you can describe a page with <meta name="description" content="Description of the page" or keywords etc. here are a couple of examples:
<meta charset="UTF-8">
<meta name="description" content="Free Web tutorials">
<meta name="keywords" content="HTML,CSS,XML,JavaScript">
<meta name="author" content="John Doe">
With html5 you can also set the viewport over the meta tag. You can simply type <<meta name="viewport" content="width=device-width, initial-scale=1.0">. It basically allows you to e.g. resize the text if your device is a smartphone or smth
